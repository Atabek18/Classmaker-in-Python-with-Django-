Learn how charging for Tests and Fees work?

PayPal.com is used to take payments.

Test taker (Payee) steps

  1. Test takers visit the Test page which will display the Payment instructions you set above, the price you set and a Continue button.
  2. Test takers add their Email and a Password which they can use to re-log into the Test should they have any computer or network issues during the Test.
  3. Test takers Name and any "Extra information" Questions (which can optionally be set below) are then collected.
  4. A PayPal payment Popup window will then open where they can pay with a Credit card or a PayPal account. (No references to ClassMarker are on the PayPal payment pages.)
  5. Payment is made. (Payees do not require a PayPal account).
  6. After payment is made, Test takers are prompted to Close the PayPal payment Popup window and go back to the original page where they can continue to the Test.

Troubleshooting

If for any reason after making a successful Payment, Test takers cannot continue to their Test, they have two options:

  • Refresh their browser window and select Continue again to Start the Test.
  • Go back to the "Test Start Link" and use the same Email address and Password they used previously before Paying for the Test. This email address will be detected as "Paid" and waiting to take the Test and they can continue to the Test without being prompted to pay again.

Test takers PDF guide

Here is a PDF Guide for Test takers with information and troubleshooting tips which you can download and provide to them should they need it.

Restrictions

All restriction settings such as "Access lists", "Allowed attempts" and "Password" will still work as normal and must be passed before a Payment can be made.

For example, if you set a Password in the Restriction settings above, Test takers will first be asked for the password before they can continue, pay for and take your Test.

How do I receive payments?

A PayPal Premier or Business account is required.
Also ensure your PayPal account is PayPal CIP Approved.
Contact Paypal.com for more information regarding Premier or Business accounts.

Add and verify the email address you have registered with your Paypal.com account via your My account page.

Money will be paid to your PayPal.com account instantly upon purchase.

Your PayPal.com account must accept the currency you are charging in.

Minimum charge you can set is: $1.95 USD (or equivalent)

Credit usage & Fees

When a Test is paid for by a Test taker, the first 10 attempts they take (if you allow more than one attempt) will not use credits in your account. (Set Attempts allowed above under the "Attempts" setting). Credits will only be used after 10 attempts per paid Test taker.

If multiple attempts are allowed, Test takers can go to the Test Link again and use the Email and Password they registered on the "Test start page" to resume another attempt.

ClassMarker Fees: 10% of purchase price (Minimum fee: $1.00 USD per purchase)

PayPal Fees: PayPal purchasing fees are paid for in full by ClassMarker for each purchase.

Require ID:Send User IDs from your system (website) to ClassMarker.

This setting is optional & may require a web developer.

Settings & Instructions

You can send a unique id for each of your Test takers to be saved in ClassMarker against their Test results.

The cm_user_id can be used to:

  • Track Test takers from your system by ID
    EG: ...cm_user_id=123

  • Track where a user came from on your website to access your Test
    EG: ...cm_user_id=course7

  • Combined tracking: The ID you send could be a combination of user_id-course_id
    EG: ...cm_user_id=123-765

cm_user_id can be any alphanumeric string under 100 characters long.

Your web developer needs to upgrade your system to append the ID with the Test link automatically for each user, using a POST or GET parameter: "&cm_user_id=1234".

Documentation for Web developers >>

Note: Even if this option is not turned on, if a cm_user_id is used, ClassMarker will still save these for you automatically. This means you can have Test takers with or without a cm_user_id parameter take your Tests.

Important: If you are not sure about these options do not check these boxes as your Tests will not display to your Test takers if not implemented correctly.

Tip

Append 'Test results' parameters to return link.

Advanced

Note: 'Webhooks' below is the best method to have Test results returned and recorded on your website in real time.

You will still use this 'Return link' setting to direct users back to a specific page on your website after the Test is finished, however, consider using Webhooks to pass results pack to your website rather then append them here.

1. Default parameters
By default, if a cm_user_id or Access list item are used to access your Test, the cm_user_id and/or Access list item cm_access_list_item parameters will automatically be appended to your return LINK for you to record and use with your system.

Example:
yourReturnLink?cm_user_id=1234&cm_access_list_item=5678

2. Add your own unique parameters
You can also add your own unique parameters to be returned to you, by prefixing the parameter names with cm_return_ and sending them to ClassMarker on the Test link

Example:
The parameter cm_return_employee_id=1234 will be returned to you as &cm_return_employee_id=1234

3. Append \'Test results\' parameters
You can also choose to append the actual Test points, Test takers name, test duration and email address (by checking the checkbox below) to the Return Link so you can record these details on your own website in real time when a user clicks the Return Link to your website.

Example:
yourReturnLink?cm_ts=30&cm_tsa=40.0&cm_tp=75.0&cm_td=0-12-35&
cm_fn=John&cm_ln=Smith&cm_e=john@example.com

Where:
cm_ts=Test Score (Points scored)
cm_tp=Test Percentage
cm_td=Test Duration (0-12-35 = 0h 12m 35s)
cm_fn=first name
cm_ln=last name
cm_e=email address

Notes:
1) At present cm_ts and cm_tp can be to given to one decimal place if applicable.
EG: 50 or 50.1. Please ensure your system will accept a value to 1 decimal place, 50.0 or 50.1 for future changes.
(For now convert any whole number to one decimal place on your end: 50 to 50.0).

2) In a future enhancement we will be adding an extra parameter cm_tsa=Test Score Available which will be (Total points available) for the Test.
(For now check is this parameter exists and ignore otherwise).
